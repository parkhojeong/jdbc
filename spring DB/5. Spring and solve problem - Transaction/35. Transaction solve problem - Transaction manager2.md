- Transaction manager(DataSourceTransactionManager) flow
* Transaction start
1. service logic start - transactionManager.getTransaction()
2. TransactionManager generate connection using DataSource
3. connection.setAutoCommit(false) - start transaction
4. store connection in TransactionSynchronizationManager
5. TransactionSynchronizationManager store using ThreadLocal

* logic start
6. call repository method
7. repository method get connection from DataSourceUtils.getConnection()
    - from TransactionSynchronizationManager
    - use same connection. so, keep transaction
8. execute query using connection

* Transaction end
9. service do transactionManager.commit() or .rollback().
10. get connection from TransactionSynchronizationManager
11. commit or rollback transaction using connection
12. release resources
    - clear TransactionSynchronizationManager(including ThreadLocal)
    - connection.autoCommit(true) for connection pool
    - connection.close() - if connection pool is used, return connection to pool
