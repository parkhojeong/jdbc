client connection(1) - db connection(1) - db session(1)
- db session 
  - manage transaction
  - run sql in transaction

auto commit
- if auto commit is true, commit automatically after each sql
- if auto commit is false, commit or rollback manually
  - auto commit is false, means "start transaction"

DB lock
- session get lock before change(update, delete)
  - another session wait until lock is released for change
- if get lock during select, use select for update
- after commit or rollback, lock is released

transaction
- server application start transaction before business logic
- application use same connection for db transaction
  - same connection is used for all db operations
